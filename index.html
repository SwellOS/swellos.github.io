What is SwellOS?

An independent Linux distribution built from the ground up — not a fork or derivative of any existing distro.

SwellOS follows the Linux From Scratch philosophy: every component is compiled from source, optimized for performance, and hand-picked. The result is a lean, fast system with no bloat.

Now in alpha — 14 milestones complete. The full package catalog (59 scripts across 8 categories) and the swell-build toolchain are ready. Bootable ISO generation is included. Binary package repository infrastructure and documentation book are live.

sbpm now supports freeze/unfreeze/rollback (per-package upgrade prevention with version pinning), build (source compilation via swell-build), and a comprehensive docs book covering the full build pipeline from kernel to desktop.

Key features

What makes SwellOS different from other distributions.

⚡

Performance tuned

Preemptible kernel (CONFIG_PREEMPT), tickless idle (NO_HZ_FULL), schedutil governor, ZSTD kernel compression — optimized for desktop responsiveness.

🧱

Monolithic kernel

No module support (CONFIG_MODULES=n). All drivers compiled in. NVMe, SATA, virtio, e1000, USB, ext4, btrfs, vfat — all built-in.

📦

sbpm + swell-build

Custom Rust toolchain: sbpm for binary package management (install, freeze/rollback, build), swell-build for source builds and ISO generation.

🔄

runit init

Fast service supervision: one-time startup (stage 1), supervised services via runsvdir (stage 2), clean shutdown (stage 3).

🏗️

From scratch

Not based on Arch, Debian, or any existing distro. Every package cross-compiled from source for SwellOS using the host toolchain.

🌐

59 package scripts

Core toolchain, desktop environments (KDE Plasma 6, Hyprland, i3, WindowMaker), browsers, dev tools, gaming, multimedia, office, network.

Build your own

SwellOS is built entirely from source. Here's how the build pipeline works.

# 1. Clone the meta-repo
$ git clone https://github.com/SwellOS/swellos.git
$ cd swellos && git submodule update --init --recursive

# 2. Build sbpm (package manager)
$ cd sbpm && cargo build --release

# 3. Build the kernel
$ cd kernel && ./scripts/build.sh

# 4. Build all packages
$ cd ../swell-build && cargo build --release
$ ./target/release/swell-build world -j $(nproc)

# 5. Generate repo index
$ ./target/release/swell-build repo

# 6. Build a single package via sbpm
$ sbpm build core/sed

# 7. Rollback: freeze packages to pin versions
$ sbpm -F glibc gcc # freeze specific (rollback protection)
$ sbpm -u glibc # unfreeze specific

# 8. Build bootable ISO
$ ./target/release/swell-build iso -K /path/to/bzImage -f

# 9. Test in QEMU
$ qemu-system-x86_64 -m 2G -cdrom /usr/src/swell/repo/swellos-0.1-x86_64.iso -cpu host -enable-kvm -serial stdio

All 59 packages are build scripts in the packages repo. Dependencies are resolved automatically by swell-build world. The iso subcommand requires mksquashfs, xorriso, grub-mkrescue, and a pre-built busybox binary on the host.
